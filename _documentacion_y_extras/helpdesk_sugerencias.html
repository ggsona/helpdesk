📋 Sugerencias de Mejora para Helpdesk Laravel

Este documento contiene sugerencias específicas para mejorar y añadir valor operativo al sistema de Mesa de Ayuda (Helpdesk GDC), incluyendo hallazgos de la comparación con el módulo de Incidencias de SIGEINV.

🎨 A. Mejoras Estéticas y de Experiencia de Usuario (UX/UI)

A1. Dashboard con Datos Reales y Gráficos Interactivos

Estado actual: El dashboard muestra tarjetas con datos hardcodeados (12, 25, 8, 45m) y los placeholders de gráficas están vacíos.

Propuesta: Conectar las tarjetas a datos reales con queries de Eloquent. Integrar Chart.js o ApexCharts para las gráficas de "Rendimiento por Técnico" (barras horizontales) y "Porcentaje por Categoría" (donut chart).

A2. Animaciones Micro en Tarjetas y Tablas

Estado actual: Las tarjetas .card-premium tienen estilos completos y soportan dark mode, pero carecen de animaciones al interactuar.

Propuesta: Agregar transiciones suaves de transform y box-shadow al hacer hover sobre las tarjetas y filas de tabla.

.card-premium {
    transition: transform 0.2s ease, box-shadow 0.2s ease;
}
.card-premium:hover {
    transform: translateY(-2px);
    box-shadow: 0 8px 25px rgba(0,0,0,0.08);
}

A3. Indicadores de Estado con Colores Semánticos (Badges Mejorados)

Propuesta: Crear badges con gradientes sutiles y bordes redondeados tipo "pill" que sean más expresivos para los estados de los tickets.

A4. Skeleton Loaders para Carga

Propuesta: Cuando las tablas o tarjetas están cargando datos, mostrar esqueletos animados (shimmer effect) en lugar de pantalla en blanco. Esto da una sensación profesional y de alta calidad.

A5. Breadcrumbs Contextuales

Estado actual: El header superior está casi vacío.

Propuesta: Agregar migas de pan automáticas: Dashboard > Mesa de Despacho > Ticket #42.

A6. Editor de Texto Enriquecido y Pegado de Imágenes Recuperada

Estado actual: Los campos de comentario son <textarea> de texto plano. Para mostrar un error con captura, el usuario debe adjuntar un archivo manualmente.

Propuesta: Integrar un editor ligero como TinyMCE, Quill.js o Trix en el campo de comentarios del chat. Funcionalidades clave:

  • Pegado de imágenes desde portapapeles (Ctrl+V): Screenshot → Ctrl+V → subida automática vía AJAX.
  • Formato básico: Negritas, cursivas, listas y bloques de código.
  • Previsualización inline: Capturas pegadas visibles directamente en la burbuja del chat.
Especialmente crítico para soporte técnico donde "una imagen vale más que mil palabras".

A7. Chat Flotante Fijo (Panel Deslizable) Prioridad Alta

Estado actual: El chat está embebido dentro del flujo vertical de show.blade.php. El técnico pierde de vista los detalles del ticket al hacer scroll hasta el chat.

Propuesta: Convertir el chat en un panel flotante lateral fijo que se desliza desde la derecha (estilo WhatsApp Web / Intercom).

Comportamiento:

  • Botón flotante (FAB) en la esquina inferior derecha: 💬 con badge de número de mensajes.
  • Panel lateral deslizable (400px) con header, burbujas de chat scrolleables y footer fijo con textarea + botón de enviar.
  • Siempre visible mientras el técnico navega por detalles, adjuntos o timeline.
  • En móviles: ocupa el 100% de la pantalla como modal fullscreen.
  • Se cierra con botón ×, tecla Escape o clic fuera (backdrop).
.chat-floating-panel {
    position: fixed; right: 0; top: 0;
    width: 400px; height: 100vh;
    background: var(--bs-body-bg);
    border-left: 1px solid var(--bs-border-color);
    box-shadow: -8px 0 30px rgba(0,0,0,0.12);
    z-index: 1050;
    display: flex; flex-direction: column;
    transform: translateX(100%);
    transition: transform 0.35s cubic-bezier(0.4,0,0.2,1);
}
.chat-floating-panel.open { transform: translateX(0); }
.chat-fab {
    position: fixed; bottom: 28px; right: 28px;
    width: 56px; height: 56px; border-radius: 50%;
    background: linear-gradient(135deg, #0d6efd, #4f46e5);
    color: white; border: none;
    box-shadow: 0 6px 20px rgba(13,110,253,0.4);
    z-index: 1040; cursor: pointer;
}
.chat-fab:hover {
    transform: scale(1.08);
    box-shadow: 0 8px 28px rgba(13,110,253,0.55);
}
Ventaja clave: El técnico puede leer detalles del equipo, ver adjuntos y responder sin perder contexto. El chat se siente como una app de mensajería moderna.

⚙️ B. Mejoras de Funcionalidad y Automatización

B1. Cronómetros Visuales de SLA Prioridad Alta

Estado actual: Los tickets tienen Prioridad pero no hay indicador visual del tiempo transcurrido ni SLA.

Propuesta: Crear una barra de progreso visual por cada ticket que muestre el tiempo restante según su prioridad:

  • Baja: 72 horas → Barra verde
  • Media: 48 horas → Barra amarilla al 50%
  • Alta: 24 horas → Barra naranja al 70%
  • Crítica: 4 horas → Barra roja si excede

B2. Respuestas Rápidas / Macros para Técnicos Prioridad Alta

Estado actual: Los técnicos escriben cada comentario manualmente.

Propuesta: Crear una tabla respuestas_rapidas con columnas titulo, contenido, id_usuario_creador. En la interfaz del chat, agregar un botón "⚡ Macro" que despliegue un dropdown con las plantillas guardadas.

B3. Línea de Tiempo del Ticket (Timeline)

Propuesta de SIGEINV: En SIGEINV cada incidencia registra automáticamente quién la creó y modificó gracias a spatie/laravel-activitylog.

Implementación: En la vista show.blade.php del ticket, agregar un panel lateral tipo "timeline" vertical que muestre cada evento del ticket cronológicamente.

B4. Encuestas de Satisfacción (CSAT)

Propuesta: Cuando el técnico cierra un ticket, mostrar al usuario un modal de calificación con 5 estrellas. Guardar en tabla ticket_calificaciones. Esto alimenta directamente la vista de Rendimiento Técnico.

B5. Base de Conocimientos (Deflexión de Tickets)

Estado actual: Existe un modelo Faq.php pero parece sin uso.

Propuesta: Activar y llenar esa tabla. Sugerir automáticamente artículos de FAQ según la categoría seleccionada al crear un ticket.

B6. Auto-asignación Inteligente (Round-Robin)

Propuesta de SIGEINV: En SIGEINV las incidencias se filtran por especialidad_id del técnico.

Implementación: Crear un botón "⚡ Auto-asignar" que busque al técnico con menos tickets activos.

B7. Ficha de Servicio en PDF (Acta de Entrega)

Propuesta de SIGEINV: SIGEINV tiene fichas PDF individuales por incidencia.

Implementación: Crear una ruta que genere un PDF con datos del usuario, equipo, problema, solución y espacio para firma. Se usa barryvdh/laravel-dompdf que ya está instalado.

B8. Exportación Masiva de Tickets a Excel

Propuesta: Permitir al Gestor exportar la tabla de tickets filtrada a Excel con maatwebsite/excel.

B9. Fusión de Tickets (Merge)

Propuesta: Permitir al Gestor fusionar múltiples tickets idénticos durante fallas masivas en un "Ticket Padre".

B10. Creación de Tickets por Email

Propuesta: Configurar un comando de Laravel que lea una bandeja de entrada IMAP y cree tickets automáticamente.

B11. Notificaciones Webhooks a Microsoft Teams / Slack Recuperada

Estado actual: No existe un sistema de notificaciones externo.

Propuesta: Al crear un ticket de prioridad Alta o Crítica, enviar automáticamente un Webhook a Teams/Slack con un mensaje formateado:

🚨 Nuevo Ticket Crítico #142
Usuario: Juan Pérez (Departamento de RRHH)
Categoría: Hardware
Asunto: Laptop no enciende
Prioridad: 🔴 Crítica
→ Ver ticket: https://helpdesk.tuempresa.com/soporte/tickets/142

En Laravel se implementa con Http::post() al URL del Webhook. Alerta a todo el equipo de TI inmediatamente.

🔀 D. Hallazgos de la Comparación con SIGEINV

D1. Reactividad con Livewire (Diferencia Clave)

SIGEINV usa Livewire 3, logrando filtrado y paginación instantáneos. Helpdesk usa controladores tradicionales. A mediano plazo, considerar migrar la Mesa de Despacho a Livewire.

D2. Vinculación Polimórfica con Activos

SIGEINV vincula incidencias polimórficamente a múltiples tipos de activo. El modelo de Helpdesk (tabla única Equipo) es más limpio para soporte. Considerar agregar campos como IP o AnyDesk ID al equipo.

D3. Configuraciones Dinámicas desde la BD

Ampliar las configuraciones para incluir tiempos de SLA, toggle de auto-asignación y de encuesta de satisfacción.

D4. Filtros Avanzados por Especialidad

Agregar una relación entre Categoria y User (muchos a muchos) para que el técnico de "Hardware" reciba solo tickets de esa categoría.

D5. Notas Internas y Visibilidad Diferenciada (Ventaja de Helpdesk)

El sistema de chat con notas internas (checkbox es_interno) y burbujas diferenciadas es una funcionalidad que SIGEINV NO tenía. Mantener esta ventaja.

D6. Sistema de Auditoría

Evaluar si vale la pena migrar a spatie/laravel-activitylog para reducir código boilerplate y tener logs automáticos.
